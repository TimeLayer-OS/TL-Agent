# TL-Agent

[English](README.md) · **Русский**

**Агент не может разрешить себе действие сам. И не может переписать, что сделал.**

TL-Agent даёт ИИ-агенту разрешения в виде нотариальных квитанций сети [TimeLayer](https://timelayer-os.com) — квитанций, которые он не способен выписать себе сам. Каждое разрешённое действие подкреплено одной из них. История агента неподделываема и проверяется офлайн любой третьей стороной.

> **НЕТ КВИТАНЦИИ — НЕТ ДЕЙСТВИЯ**

---

## Почему это, а не «просто проверки в коде агента»

| Сам (флаги + лог в коде) | TL-Agent |
|---|---|
| Агент по сути сам решает — может «разрешить» себе обойти проверку | Разрешение — **нотариальная квитанция, которую агент не выпускает сам** (INV-01) |
| История действий — лог у оператора, редактируем | История — квитанции, **которые нельзя переписать**, проверка третьей стороной |
| Доверие — к твоему коду и серверу | Доверие — к **кворуму независимых операторов**, офлайн-проверка открытым кодом |
| «Сделано» — на слово модели | «Сделано» — только при наличии квитанции (INV-06: текст модели не доказательство) |

**Одной строкой: проверку в коде агент обходит; нотариальную квитанцию, которую он не может выписать себе, — нет.**

---

## Три гарантии

- **Контроль действий.** Каждое действие проходит через гейт: нет валидной квитанции — действие не выполняется. Топология задаёт, что вообще позволено.
- **Неподделываемая история.** Что агент сделал — записано квитанциями, которые ни агент, ни оператор не могут переписать. Проверяется офлайн третьей стороной.
- **Fail-closed по замыслу.** Любой конфликт, отсутствие разрешения, неизвестное действие → агент останавливается и ждёт человека. Молчание — безопасный отказ, а не скрытый успех.

## Быстрый старт

### 1. Скачать `tl-agent`

Скачать бинарник из [Releases](../../releases/latest).

```bash
chmod +x tl-agent
./tl-agent --help
```

### 2. Гейт-проверка действия перед выполнением

```bash
tl-agent check ./example-bundle action_read_files
# ALLOW
#   action  : action_read_files
#   receipt : tlr_20260624_000001
#   next    : action_summarize
```

Exit code `0` = ALLOW, `1` = STOP. Подключается к любому циклу агента или shell-скрипту.

### 3. Проверить весь bundle

```bash
tl-agent audit ./example-bundle
# === tl-agent audit ===
# actions : 4  valid: 4  stop: 0
# result: ALL VALID — bundle is ready for use
```

### 4. В Rust-агенте

```rust
let bundle = AgentBundle::load("./agent-bundle")?;

match bundle.check_action("action_read_files") {
    CheckResult::Allow(receipt) => {
        // выполнить — разрешение нотариально подтверждено
    }
    CheckResult::Stop { reason, .. } => {
        // остановиться — NO_RECEIPT, TOPOLOGY_VIOLATION, SIGNATURE_INVALID и т.д.
    }
}
```

## Собрать bundle

Самый быстрый способ — **[Конструктор агента](https://cabinet.timelayer-os.com/agent)** в личном кабинете TimeLayer:

1. Задайте действия и топологию визуально
2. Каждое действие получает квитанцию, нотаризованную сетью
3. Скачайте готовый ZIP-bundle

Или создайте структуру bundle вручную по `SPEC.ru.md`.

## Структура bundle

```
agent-bundle/
  manifest.json          ← bundle ID, владелец, кол-во квитанций, tl_agent_version
  topology.json          ← допустимые переходы между действиями
  policies/
    agent_policy.json    ← что агенту запрещено делать
    tool_policy.json     ← какие инструменты разрешены
    stop_policy.json     ← когда агент обязан остановиться
  receipts/
    <action_id>/
      envelope.json      ← метаданные действия, allowed_next_actions, ссылка на квитанцию
      cert.tlcert        ← нотариальный сертификат — никогда не изменяется
      bundle.tlbundle    ← нотариальный бандл (подписи) — проверяется вместе с cert
  exports/
    segment_01/          ← сегмент bundle для выдачи агенту
```

`cert.tlcert` + `bundle.tlbundle` выпускаются сетью TimeLayer. TL-Agent только читает и верифицирует их — никогда не записывает и не изменяет.

## Справочник CLI

```
tl-agent check  <bundle_dir> <action_id>              Гейт-проверка. Exit 0=ALLOW, 1=STOP.
tl-agent next   <bundle_dir> <action_id>              Список допустимых следующих действий по топологии.
tl-agent audit  <bundle_dir>                          Проверка каждого действия в bundle.
tl-agent record <bundle_dir> <action_id> <sha256>     Запись выполнения в execution_log.jsonl.

Опции:
  --verifier <path>    Путь к бинарнику timelayer-verifier.
                       Авто-поиск: рядом с tl-agent → bundle/bin/ → PATH
```

## Модель безопасности

| Инвариант | Правило |
|-----------|---------|
| INV-01 | Агент не выпускает валидные квитанции сам |
| INV-02 | Агент не изменяет bundle пользователя |
| INV-03 | Каждое действие требует `permission_receipt` |
| INV-04 | Каждый переход проверяется по `topology.json` |
| INV-05 | Любой конфликт → STOP |
| INV-06 | Текст модели не считается доказательством |
| INV-09 | Квитанция (`cert.tlcert` + `bundle.tlbundle`) не модифицируется никогда |

**Fail-closed**: любая ошибка (нет квитанции, невалидная подпись, неизвестное действие) → STOP, никогда не молчаливый пропуск.

## Честная рамка

- **Сеть TimeLayer**: кворум независимых операторов, публичные ключи на GitHub
- **Подписи**: Ed25519; хеш-обязательство квантово-устойчиво; постквантовые подписи — в дорожной карте
- **Что такое TL-Agent**: guardrails + неподделываемый аудит для кооперативного агента — не песочница, физически запирающая злонамеренный код (кроме air-gapped режима, и то от агента, не от хоста)
- **Не память в смысле recall** — провенанс. Квитанции не хранят содержимое и не делают смысловой поиск
- Внешний аудит сети: в дорожной карте
